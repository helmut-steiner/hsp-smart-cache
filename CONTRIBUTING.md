# Contributing

## Development

1. Fork the repo.
2. Create a feature branch.
3. Make changes.
4. Run WordPress and verify via Settings → HSP Smart Cache → Run Cache Tests.
5. Submit a PR.

## Coding Standards

- Follow WordPress coding standards.
- Keep changes minimal and focused.

## Reporting Issues

Include:

- WordPress + PHP versions
- Plugin version
- Steps to reproduce
- Error logs if available
